# 校园互助服务平台（CampusHub）

CampusHub 是一个面向大学校园场景的互助服务平台，目标是为学生提供统一的信息发布与协作入口，覆盖快递代取、学习辅导、二手物品交换、活动组队等常见互助需求。相比需求分散在微信群、QQ 群、贴吧等渠道的方式，平台希望提升信息组织效率、供需匹配效率以及协作过程中的可信度。

## 项目简介

根据课程总览文档，CampusHub 面向三类核心角色：

- 需求方：发布互助需求的学生
- 服务方：接单并提供服务的学生
- 管理员：负责内容审核、用户管理和数据统计

项目围绕以下核心业务展开：

- 用户注册、登录与个人资料管理
- 互助需求发布、分类、浏览、筛选与详情查看
- 订单流程管理，包括接单、确认、进行中、完成和评价
- 评价与信用体系，支持双向评分和信用展示
- 消息通知、管理后台、推荐匹配等扩展能力

课程要求中的最小可行产品范围重点包括：

- 用户管理
- 需求发布与浏览
- 订单流程管理

## 技术栈
结合当前项目规划，当前已确认的技术栈方向如下：

- 前端：React Native、Flutter
- 后端语言：Java
- 工程构建：Maven
- 数据库：MySQL
- 开发环境：IntelliJ IDEA 
- 持续集成：GitHub Actions

## 仓库结构

当前仓库的实际结构如下：

```text
Project/
├─ .github/
│  └─ workflows/
│     └─ ci.yml
├─ Docs/
│  ├─ p0/
│  │  ├─ p0.md
│  │  └─ AI协作契约.md
│  └─ p1/
│     └─ p1.md
├─ src/
│  └─ 开发区.md
├─ tests/
│  └─ 测试区.md
├─ .gitignore
└─ README.md
```

各目录用途说明：

- `Docs/`：课程阶段文档主目录，按阶段存放项目材料
- `Docs/p0/`：P0 阶段文档，包括项目启动说明和 AI 协作契约
- `Docs/p1/`：P1 阶段文档目录，当前用于需求分析阶段内容沉淀
- `src/`：开发内容目录，后续用于存放项目源码与实现说明
- `tests/`：测试内容目录，后续用于存放测试用例、测试脚本或测试记录
- `.github/workflows/`：仓库自动化配置目录，当前包含 CI 工作流
- `README.md`：自述文件

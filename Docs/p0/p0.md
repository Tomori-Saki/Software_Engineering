# AI 工具选型评估表

## 1. AI 编程助手 (AI Coding Assistant)
| 候选工具                 | 评估结果            | 选择/不选择理由                    |
|:---------------------|:----------------|:----------------------------|
| **Codex**            | [√] 选中 / [ ] 弃用 | 理由：逻辑生成精准，通过插件深度集成 IDE，显著提升代码编写效率与流畅度。      |
| **Cursor**           | [√] 选中 / [ ] 弃用 | 理由：原生支持文件内直接补全与对话，代码审查直观高效，极大简化了重构流程。 |
| **Google AI Studio** | [ ] 选中 / [√] 弃用 | 理由：目前仅提供网页版交互，缺乏 IDE 原生插件支持，导致代码审查与同步不便。             |
| **其他 (如通义灵码)**       | [ ] 选中 / [√] 弃用 | 理由：模型综合理解能力与代码补全的准确性尚有差距，难以满足复杂业务开发需求。                     |

## 2. AI 对话工具 (AI Chat Tools)
| 候选工具          | 评估结果            | 选择/不选择理由                                   |
|:--------------|:----------------|:-------------------------------------------|
| **ChatGPT**   | [ ] 选中 / [√] 弃用 | 理由：免费额度限制较为严格，且高级订阅成本较高，个人及小团队使用的性价比优势不明显。 |
| **Gemini**    | [√] 选中 / [ ] 弃用 | 理由：多模态理解力强，且提供丰富的免费功能与较大的额度，综合表现最优    。    |
| **Deepseek**  | [ ] 选中 / [√] 弃用 | 理由：多模态识别支持有限，且大文件处理能力较弱，场景覆盖不全      。      |
| **其他（如文心一言）** | [ ] 选中 / [√] 弃用 | 理由：在逻辑推理和语义理解的深度上稍逊一筹，回答质量无法稳定对标国际主流模型。                                    |

## 3. AI 代码审查工具 (AI Code Review)
| 候选工具                    | 评估结果           | 选择/不选择理由                                                           |
|:------------------------|:---------------|:-------------------------------------------------------------------|
| **CodiumAI (PR-Agent)** | [√] 选中 / [] 弃用 | 理由：免费，可以自动生成 PR 摘要、提供代码改进建议、检测潜在逻辑错误，并直接在 GitHub/GitLab 的评论区与开发者互动 |
| **Qodana (JetBrains)**  | [] 选中 / [√] 弃用 | 理由：需要在 CI/CD 配置文件中进行详细设置，对非 JetBrains 环境的支持度低，并且收费                 |

## 4. AI 测试工具 (AI Testing Tools)
| 候选工具 | 评估结果            | 选择/不选择理由                                   |
| :--- |:----------------|:-------------------------------------------|
| **CodiumAI** | [√] 选中 / [ ] 弃用 | 理由：不仅生成覆盖率，还能分析代码逻辑并建议边界测试和负面测试，并且免费版不限制功能 |
| **Diffblue Cover** | [ ] 选中 / [√] 弃用 | 理由：仅支持java，对免费版有使用限制，仅擅长回归测试，其他部分相比另一款较差   |
